# Артефакт - Декомпозиция AI-системы для оценки (Прототип)

## 1. Декомпозиция на данном этапе разработки:

**Вход:**
На вход дается текстовый запрос пользователя (преимущественно на английском языке для корректного поиска имен в базе данных). На данном этапе разработки решает задачу поиска подробных статистик игрока и ее аналитики. Формирование первичного скаутского вердикта о целесообразности трансфера или наблюдения за игроком.

**Процесс:**

1. **Классификация:** Агент принимает пользовательский запрос (в виде промпта) и принимает решение об использовании инструментов для дальнейшей работы по запросу.
2. **Выполнение инструментов:** Агент выбирает один из следующих инструментов для работы:
    * **Searchplayer ID tool:** Если в запросе указано имя игрока, агент использует этот инструмент для нахождения уникального ID в базе SofaScore.
    * **Stats tool:** Получение актуальных игровых метрик (голы, ассисты, xG, рейтинг) за текущий игровой цикл (сезон). Позволяет оценить «текущую форму» игрока на основе последних матчей.
    * **Team tool (на этапе разработки):** Если запрос касается конкретной команды, а не игрока, агент ищет ID клуба, запрашивает состав клуба и итерируется по игрокам.
    * **Full stats tool:** Инструмент, позволяющий агенту получить полную статистику игрока по всем возможным метрикам, включая xG, отборы, успешный дриблинг и т.д.
3. **Обработка и фильтрация данных (в основном внутри инструментов):** Внутри инструментов система получает объемные JSON данные, которые очищаются от избыточных данных, используя лишь нужную информацию, оптимизируя таким образом работу LLM.
4. **Генерация аналитического вывода:** LLM сопоставляет критерии пользователя с полученными данными и формирует текстовые аналитические ответы.

**Выход:**
Валидный JSON, содержащий основные физические и игровые параметры игрока. А также скаутский отчет в виде текстовой интерпретации данных в виде рекомендаций и сильных/слабых сторон, а также финальный вердикт по совершению трансфера или наблюдению за игроком.

## 2. Определение компонентов для оценки

1. **Качество классификации запроса:** Оценка насколько система правильно понимает запрос и подбирает инструменты для его исполнения.
2. **Фактическая точность:** Оценка соответствия численных значений в финальном отчете аналитики, полученных напрямую из API, с фактическими данными.
3. **Надежность извлечения сущностей:** Способность системы правильно определять и искать ID команды или игрока при разных написаниях имен и названий.
4. **Стабильность формата:** Оценка количества успешных ответов, прошедших валидацию через Pydantic-схемы.

## 3. Уровни оценки

1. **Промежуточный уровень:** В качестве промежуточной оценки рассматриваем работу агента с подбором инструментов. Рассматриваем работу самих инструментов и валидный вывод — ответ каждого из них.
2. **Сквозной уровень:** В качестве сквозной оценки рассматривается вся цепочка от полученного запроса до финального ответа. Соответствует ли полученный ответ данному запросу.
