## Задание — Schema-Guided Reasoning

В ходе выполнения задания была реализована концепция **Schema-Guided Reasoning (SGR)**. Основная ее цель — не только получение структурированного вывода, а также проектирование цепочки рассуждений внутри JSON-схемы. Это позволило превратить LLM из текстового генератора в контролируемый аналитический модуль с предсказуемой логикой. 

Я разделил эту задачу на несколько структурированных выводов (**Structured Output**), как бы на уровни:

* **PlayerReasoning** работает внутри объекта каждого игрока. Вместо того чтобы сразу переходить к вердикту, модель обязана выполнить жестко заданный алгоритм: извлечь конкретные факты из API, провести бенчмаркинг (сравнение со средними показателями по лиге) и обосновать тактическое соответствие. Это гарантирует, что итоговый флаг `is_valid` является логическим следствием предыдущих шагов, а не галлюцинацией. 
* **ScoutProjectReport** отвечает за финальный синтез данных. Здесь модель сопоставляет всех найденных кандидатов между собой, формулирует логику сравнения и обосновывает выбор «победителя».

Каждое поле снабжено необходимым описанием через `description`, которое является мини-инструкцией, что избавляет модель от необходимости интерпретировать абстрактные задачи. 

И несмотря на сложность структуры, цепочка рассуждений генерируется за **один вызов LLM**. Модель видит свои предыдущие рассуждения в контексте и использует их для формирования финальных выводов. Использование такой схемы делает работу AI-агента полностью прозрачной. Любой вывод (например, рыночная рекомендация) может быть мгновенно проверен по цепочке промежуточных полей рассуждения.
