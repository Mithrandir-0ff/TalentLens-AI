# Артефакт - Оценка качества агентов

1. Сценарии
* Сценарий точечного поиска игрока. Пользователь запрашивает определенного игрока и возможно какие-то критерии, по которым нужно сделать анализ, после чего система с помощью инструментов делает этот анализ.
* Сценарий скаутской работы. Пользователь запрашивает критериальный поиск игроков, к примеру по возрасту, нации и статистике передач, после чего система с помощью инструментов поиска находит кандидатов/кандидата.
* Сценарий сравнения. Пользователь запрашивает сравнение футболистов по каким-то критериям, после чего система проводит сравнительный анализ и выдает вердикт.

2. Метрики
* Среднее число шагов на задачу
* Средняя стоимость на задачу
* Среднее время выполнения на задачу

3. Данные
Входные данные предоставляются в виде xlsx таблицы с запросом и выводом той же таблице, но с указанием ответа системы, числа шагов, стоимости и времени выполнения задачи.

4. Анализ результатов и план оптимизации
Эффективность точечного поиска: Сценарии Point Search работают стабильно (4–5 шагов, ~15 сек), демонстрируя качественный анализ метрик (xG, xA).
Надежность данных: Система не галлюцинирует, при отсутствии данных в API агент корректно возвращает пустой результат с пояснением.
Проблема перерасхода ресурсов: Сложные скаутские запросы и ошибки API (429) приводят к зацикливанию агента (до 20 шагов и 100к+ токенов), так как он пытается бесконечно перефразировать запрос.

План возможных улучшений:
Лимиты итераций: Ограничение max_iterations до 7–8 шагов для предотвращения бесконечных циклов.
Обработка ошибок: Инструкция агенту немедленно прекращать поиск, если инструмент возвращает ошибку API или отсутствие данных.
Кэширование: Сохранение ID игроков и клубов в локальную базу для снижения нагрузки на API и уменьшения времени отклика.
Entity Resolution: Улучшение алгоритма сопоставления имен для более точного выбора игрока.
