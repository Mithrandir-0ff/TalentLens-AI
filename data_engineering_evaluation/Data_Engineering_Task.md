# Data Engineering Task

## 1. Реализация стратегии генерации данных

Была выбрана шаблонная генерация, для которой был написан python-скрипт `generate_dataset.py`. Использован фиксированный random seed, чтобы проверяющий мог получить те же значения на выходе.

Для шаблонной генерации были заранее определены текстовые шаблоны запросов, в которые подставляются значения измерений (сценария, лиги, позиции, возрастной группы). Концептуально пространство запросов основано на декартовом произведении измерений, но в реализации использовалась случайная выборка из этого пространства. Для каждого сценария случайным образом назначались остальные параметры до достижения заданного количества примеров.

## 2. Генерация не менее 100 примеров

В ходе генерации было создано 114 примеров. После фильтрации итоговый размер датасета составил 106 примеров, исключив дублирующие примеры. Целевые пропорции по сценариям соблюдены наиболее точно, так как их количество контролировалось явно через `target_counts`, а остальные измерения распределились вероятностно.

## 3. Проверка охвата измерений

Для проверки полноты покрытия реализован python-скрипт `validate_coverage.py`. Скрипт программно анализирует итоговый датасет и проверяет наличие всех категорий в указанных измерениях:

- тип сценария
- позиция игрока
- лига
- уровень сложности
- возрастная группа

Проверка дает гарантию, что ни одна из категорий не была полностью пропущена в итоговом датасете.

## 4. Обработка данных

В процессе генерации реализована базовая обработка данных:

- проверка минимальной длины запроса (не менее 5 слов)
- проверка наличия ожидаемого набора инструментов
- формирование структурированного JSON-файла с метаданными
- вычисление статистики распределений по измерениям

Все примеры сохраняются в формате JSON с полями:
`id`, `query`, `scenario`, `complexity`, `position`, `league`, `age_group`, `expected_tools`.

## 5. Удаление данных низкого качества

На этапе постобработки выполнялась фильтрация с удалением дубликатов запросов и исключением некорректных или неполных примеров. Было удалено 8 дубликатов. Доля отфильтрованных примеров составила около 7%.

В результате был сформирован воспроизводимый синтетический датасет, пригодный для оценки выбора инструментов и логики работы системы.
